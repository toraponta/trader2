# シグナル検知プラグイン要件仕様

本ドキュメントでは、AIエージェントへのコンテキスト過負荷やAPIコスト増大を防ぐために、Pythonバックエンド側で全ウォッチリスト銘柄の事前足切り（スクリーニング）を行う「シグナル検知プラグイン」の構成と仕様を定義します。

## 1. 設計思想とアーキテクチャ

*   **処理の重いAIと高速なバックエンドの分業**: 
    毎日何十〜何百という監視銘柄をすべてAI（Gemini）に分析させると、API料金が高騰し、AIの判断精度も落ちます。そのため、**「イベント（シグナル）が発生した銘柄だけをバックエンドが瞬時に抽出し、AIへの初期プロンプトにアラート文として差し込む」**構成とします。
*   **クリーンな拡張性 (ディレクトリ配置)**:
    検知ロジックはコアエンジンにハードコードせず、独立したPythonスクリプトとして `plugins/signals/` ディレクトリに配置します。

### インターフェース仕様（全プラグイン共通）
各プラグインは共通の関数（例: `check_signal(ticker, market_data, config)`）を実装し、検知結果を以下の辞書形式で返す標準仕様とします。

```python
# 戻り値のイメージ
{
    "triggered": True,
    "signal_type": "TECHNICAL_BREAKOUT",
    "alert_text": "過去20日間の最高値をブレイクアウトしました（現在値: 1,520円）"
}
```

---

## 2. 必要な初期プラグイン一覧

要件定義を満たすために初期実装するプラグイン群です。設定パラメータ（JSON）はデータベース（`signal_plugins`）で管理され、エージェントごとに動的に変更可能です。

### 2.1 株価急変動検知プラグイン (`price_surge_plugin.py`)
*   **目的**: 短期的な資金流入やパニック売りが発生している注目銘柄の抽出。
*   **デフォルトパラメータ**: `{"threshold_percent": 3.0}`
*   **検知ロジック**: 前日終値からの株価変動率が `±threshold_percent`（例: ±3%）を超えた場合にシグナルを発火する。デイトレードや短期スイングを狙うエージェントに有効。

### 2.2 高値・安値ブレイクアウト検知 (`breakout_plugin.py`)
*   **目的**: 明確な上昇・下降トレンドが発生した瞬間の捕捉。
*   **デフォルトパラメータ**: `{"lookback_days": 20}`
*   **検知ロジック**: 過去 `lookback_days` 日間の最高値を上抜けた、または最安値を下抜けた銘柄を検知する。順張りのトレンドフォロー戦略エージェント向け。

### 2.3 移動平均クロス検知 (`moving_average_cross_plugin.py`)
*   **目的**: 王道のテクニカル転換点の捕捉。
*   **デフォルトパラメータ**: `{"short_period": 5, "long_period": 25}`
*   **検知ロジック**: 短期移動平均線が長期移動平均線を下から上に抜ける（ゴールデンクロス）、または上から下に抜ける（デッドクロス）状態を検知する。

### 2.4 決算発表直前アラート (`earnings_proximity_plugin.py`)
*   **目的**: 決算またぎリスクの事前回避、または決算プレイの準備。
*   **デフォルトパラメータ**: `{"alert_days_before": 5}`
*   **検知ロジック**: J-Quants等からキャッシュした決算発表予定日を参照し、発表が `alert_days_before` 日以内に迫っている銘柄に対してアラートテキストを出す。AIに「決算直前だからポジションを軽くするべきか？」の考察を促す。

### 2.5 キーワード開示検知 (`tdnet_keyword_plugin.py`)
*   **目的**: 企業のファンダメンタルズを劇的に変える適時開示（サプライズ）の即時検知。
*   **デフォルトパラメータ**: `{"keywords": ["上方修正", "自社株買い", "株式分割", "増配"]}`
*   **検知ロジック**: TDnet等の開示情報サマリーに指定キーワードが含まれる開示が出た銘柄を即座にシグナルとして抽出する。

## 3. 設定とエージェントごとのパーソナライズ

データベースのスキーマ設計に基づき、Web UI画面から以下の柔軟な制御を可能にします。

1.  **エージェントごとのプラグイン選択**:
    *   「エージェントA（短期スイング）」＝ 急変動検知（3%）とブレイクアウト検知をON。
    *   「エージェントB（中長期バリュー）」＝ 急変動検知はOFFにし、移動平均クロスと決算直前アラートのみON。
2.  **パラメータの個別チューニング**:
    *   同じプラグインでも、エージェントの性格に合わせて「変動率閾値を5%にする」など、JSONパラメータをDB上で上書き設定可能にします。

## 4. ロードマップ：AIによる完全自己学習・自己拡張機能

このプラグイン構造の究極の目的は、**「AI自身によるコード生成と機能拡張」**です。

将来的に、エージェントが自己学習バッチ処理の反省フェーズにおいて、
> *「最近、RSIが30を下回った銘柄の反発（リバウンド）を取るトレードの勝率が非常に高い。しかし現在の検知プラグインにはRSI検知がない」*

と気づいた場合、**AI自らが `rsi_rebound_plugin.py` という新しいPythonスクリプトコードを自律的に生成し、`plugins/signals/` ディレクトリに保存してデータベースに新規登録する**という、システム全体の自己進化サイクルを実現します。
